## Speech Script

### 1. 문제
내 건강 상태를 정확히 알고 관리하고 계신가요?
최근 5년간 20~30대의 당뇨병 유병률은 38%, 고혈압은 28%나 증가했습니다.
하지만 기존 건강관리 앱은 번거로운 수동 입력과 획일적인 조언에 그쳐, 청년층이 꾸준히 관리하기 어려운 한계가 있습니다.

### 2. 해결 아이디어
On-Care는 사진 한 장으로 식단을 기록·분석하고, 누적 건강 데이터를 학습한 AI 코치가 개인 맞춤 피드백을 제공합니다. 
또한 식단, 운동, 헬스장, 일정 관리까지 하나의 앱에서 통합적으로 해결합니다.

### 3. 기술 / 구현
식단 인식은 2단계 파이프라인으로 동작합니다. 
먼저 YOLOv8이 음식 여부를 판별해 불필요한 API 호출을 차단하고, Gemini Vision API가 음식 종류와 섭취량을 분석해 영양 정보를 자동 기록합니다.
AI 코칭은 RAG 기반으로 구현하고 개인 건강 데이터를 Pinecone에 저장하여, 사용자 기록을 반영한 맞춤형 답변을 제공합니다.
시스템은 FastAPI, Flutter 기반으로 구축됩니다.

### 4. MVP / 배포
현재 Gemini API 연동과 프롬프트 설계를 완료했습니다. 
MVP에서는 사진 기반 식단 기록, AI 코칭, 운동 루틴 및 캘린더 연동 기능을 제공합니다. 
앱은 iOS와 Android 동시 출시를 목표로 개발 중이며, 서비스는 Docker 환경에서 운영할 예정입니다.

### 5. 차별성
기존 헬스케어 앱들이 단순 기록에 머문다면, On-Care는 사진 기반 자동 기록과 AI 맞춤 코칭으로 실제 행동 변화까지 만듭니다.

### 4가지 핵심 차별성 (실증 근거 포함)

#### ① Vision AI 2단계 식단 인식 파이프라인
Noom·삼성헬스는 수동 입력, MyFitnessPal은 사진 인식을 제공하나 복합 요리 정확도가 낮아 수동 보정이 필요하다. 실제 20대 당뇨 환자 인터뷰 결과, 기존 앱 이탈의 핵심 원인은 "기록 노력 대비 실질적 피드백 부재"였다. On-Care는 YOLOv8(음식 여부 1차 필터) → Gemini Vision API(음식 종류·섭취량 분석) → 공공데이터 식품영양성분 DB 매핑의 3단계로 사진 한 장 완결 기록을 구현하고, 한국 음식 정확도 문제를 공공데이터로 해결한다.

#### ② RAG 기반 개인화 AI 코칭
경쟁 3사 모두 개인 건강 이력을 참조하는 AI 코칭이 없다. 인터뷰에서 확인된 바, 만성질환 환자는 ① 동일 식품 반복 섭취 시 대체 식품 정보, ② 혈당 급등·나트륨 과다 식품에 대한 맥락적 경고가 충족되지 않는다. On-Care는 LangChain + Pinecone + GPT-4o RAG 파이프라인으로 "내 이번 주 기록 기준" 맞춤 조언을 생성한다.

#### ③ 오프라인 운동 환경 연결
경쟁 3사 모두 헬스장·트레이너 연동 기능이 없어 사용자가 별도 채널로 이탈해야 한다. On-Care는 카카오맵 API 기반 헬스장 검색·예약·트레이너 인앱 채팅을 통합하고, 트레이너 연결 시 사용자의 누적 건강 데이터 요약본을 자동 전달해 초기 상담 효율을 높인다.

#### ④ 고혈압·당뇨 위험군 특화 설계
경쟁 앱들은 체중 감량 또는 일반 건강 관리 목적으로 설계되어 만성질환 관리 구조를 반영하지 못한다. On-Care는 서비스 기획 단계부터 나트륨 섭취 경고, GI(혈당지수) 기반 식품 분류, 야식·불규칙 식사 패턴 감지를 핵심 기능으로 설계하였다.

## 타겟 고객

**1차 타겟:** 20~30대 만성질환(고혈압·당뇨·이상지질혈증) 위험군
- 건강검진에서 "주의" 판정을 받았지만 아직 병원에 다니지 않는 사람
- 불규칙한 식사와 운동 부족을 인지하고 있지만 관리 방법을 모르는 사람
- 배달음식·외식 비중이 높아 직접 식단을 입력하기 어려운 직장인·대학생

**2차 타겟:** 만성질환을 이미 진단받은 2030세대
- 약 복용만으로는 불안하고, 식단 관리를 병행하고 싶지만 번거로운 사람

## Pain Point 분석

**Pain Point 1: 식단 기록의 번거로움 → 이탈의 가장 큰 원인**
현재 대부분의 앱은 음식 이름을 직접 검색해 입력하거나, 바코드가 있는 가공식품만 스캔할 수 있습니다. 한식 위주의 식사를 하는 한국인에게 "된장찌개 1그릇"을 입력하는 행위는 매끼 반복하기 어렵습니다. 사용자는 며칠 안에 앱 사용을 포기합니다.

→ On-Care 해결: 사진 1장으로 자동 분석. 입력 행위 자체를 제거.

**Pain Point 2: "나에 대한 정보가 없는" AI 조언의 무력함**
"탄수화물을 줄이세요"라는 조언은 누구나 압니다. 사용자가 원하는 것은 "내가 오늘 먹은 것 기준으로, 지금 내 상태에서 어떻게 해야 하는가"입니다. 기존 앱의 AI는 사용자 데이터를 실시간으로 반영하지 못합니다.

→ On-Care 해결: 개인 건강 이력을 Vector DB에 저장하고 RAG로 GPT-4o에 주입. "내 데이터를 아는 AI"를 구현.

**Pain Point 3: 앱 파편화로 인한 관리 포기**
운동은 A앱, 식단은 B앱, 병원 예약은 캘린더, 헬스장 정보는 네이버 검색. 이 분산된 경험이 지속적인 건강 관리를 방해합니다.

→ On-Care 해결: 식단·운동·캘린더·헬스장 검색·트레이너 채팅을 단일 앱에서 통합 제공.

**Pain Point 4: 동기 부여 유지의 어려움**
건강 관리는 결과가 느리게 나타납니다. 단기적인 보상 없이는 꾸준히 사용하기 어렵습니다.

→ On-Care 해결: 활동 포인트·Streak 보상 게이미피케이션으로 단기 동기 부여 제공.

## PMF (Product-Market Fit) 근거

국민건강보험공단 데이터 기준 20~30대 만성질환 유병률이 5년간 당뇨 38%, 고혈압 28%, 이상지질혈증 53% 증가하는 구조적 수요가 존재한다. 경쟁 제품 3사(Noom·Samsung Health·MyFitnessPal)는 모두 체중 감량 또는 일반 건강 관리 목적으로 설계되어 만성질환 위험군에 특화된 기능이 부재하다. 실제 환자 인터뷰를 통해 기존 앱이 실질적 행동 변화를 유도하지 못한다는 점이 실증되었으며, 이는 "기록 자동화 + 누적 이력 기반 맞춤 코칭 + 만성질환 특화 설계"라는 On-Care의 설계 방향이 충족되지 않은 시장 공백임을 뒷받침한다.

## 고객 인터뷰 (Customer Discovery)

**인터뷰 대상:** 20~30대 직장인 및 대학원생 3인 (건강검진 "주의" 또는 "경계" 판정 경험자)

| 구분 | 인터뷰이 A (사회초년생) | 인터뷰이 B (당뇨 전단계) | 인터뷰이 C (퍼스널 트레이너) |
|------|------------------------|--------------------------|------------------------------|
| Pain Point | "칼로리 계산을 위해 무게를 재고 검색하는 과정이 너무 번거롭다." | "병원 가이드는 너무 원론적이다. 회식 때 뭘 먹어야 할지 즉각 알려주는 코치가 필요하다." | "회원들의 식단 관리를 카톡으로 받는데, 영양 분석 데이터를 한눈에 보고 싶다." |
| 요구사항 | 사진 촬영 즉시 자동 분석 | 내 혈당/체질 데이터를 아는 AI 챗봇 | 관리 회원 데이터 요약 및 모니터링 |
| 반영 사항 | YOLOv8+Gemini 기반 자동 인식 파이프라인 | RAG(Pinecone/GPT-4o) 기반 개인화 코칭 엔진 | 트레이너 전용 데이터 요약 및 채팅 API |


**인터뷰 결과 요약:**

- "헬스케어 앱을 써본 적 있냐"는 질문에 3인 모두 "써봤지만 3일 이상 지속한 적 없다"고 답함.
- 포기 이유: "매끼 입력이 너무 번거로워서" (3인 공통), "어차피 뻔한 조언밖에 안 해줘서" (2인)
- "사진을 찍으면 자동으로 기록되는 앱이 있다면 쓰겠냐"는 질문에 3인 모두 긍정적 반응.
- "내 이번 주 식단 기록을 기반으로 AI가 코칭해준다면"이라는 가설에 "그건 진짜 유용할 것 같다"는 반응 (3인 중 2인).
- 헬스장 트레이너 연동 기능에 대해 "트레이너한테 내 식단 데이터를 자동으로 공유해주면 상담이 훨씬 효율적일 것 같다"는 의견 확인.
