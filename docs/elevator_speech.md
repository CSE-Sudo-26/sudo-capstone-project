# On-Care — Elevator Speech

> 2030 만성질환 위험군을 위해, YOLOv8+Gemini 하이브리드 비전 및 RAG 기술을 사용,
> 식단·건강 데이터의 파편화 문제를 풀어서, 나보다 내 데이터를 더 잘 아는 통합 헬스케어를 가능하게 한다.

---

## 🎤 발표 스크립트 (100초)

### 1. 문제 (20초)
20~30대 당뇨 유병률이 최근 5년간 38%, 고혈압 28% 증가했습니다.
문제는 아는데 안 바뀝니다.
기존 앱은 매번 손으로 입력해야 하고, "일반적인 조언"만 반복합니다.
**내 기록을 아는 앱이 없습니다.**

### 2. 해결 아이디어 (20초)
On-Care는 **사진 한 장**으로 식단을 기록하고,
**내 누적 건강 데이터를 아는 AI 코치**가 개인 맞춤 피드백을 줍니다.
식단·운동·헬스장·일정 관리를 하나의 앱에서 해결합니다.

### 3. 기술 / 구현 (30초)
- **Vision AI 파이프라인**: 음식 사진 → YOLOv8(음식 여부 1차 필터) → Gemini Vision API(영양 분석) → 공공데이터 식품영양성분 DB 매핑. 불필요한 API 호출을 차단해 속도와 비용을 동시에 잡습니다.
- **RAG 코칭 파이프라인**: 사용자 식단·운동·인바디 이력을 Pinecone Vector DB에 저장 → 질문마다 개인 데이터를 꺼내 GPT-4o에 컨텍스트로 주입 → "이번 주 탄수화물 12% 초과, 저녁에 닭가슴살 샐러드 추천" 같은 **내 기록 기반 답변** 생성.
- **스택**: Flutter(iOS/Android) + FastAPI(AWS EC2/Docker) + MySQL RDS + LangChain + Pinecone + FCM

### 4. MVP / 배포 (20초)
2026년 10월 Google Play·App Store 베타 등록이 목표입니다.
현재 메인 대시보드, Vision AI 식단 인식, RAG 챗봇 화면이 설계 완료됐고,
4월~9월 스프린트로 백엔드·AI 엔진·앱을 병렬 개발 중입니다.

### 5. 차별성 (10초)
Noom·삼성헬스·MyFitnessPal은 모두 **기록 도구**에 머뭅니다.
On-Care는 실제 20대 당뇨 환자 인터뷰로 확인된 네 가지 공백 —
기록 장벽·획일적 코칭·오프라인 단절·만성질환 특화 부재 — 을 동시에 해결하는 유일한 앱입니다.

### 4가지 핵심 차별성 (실증 근거 포함)

#### ① Vision AI 2단계 식단 인식 파이프라인
Noom·삼성헬스는 수동 입력, MyFitnessPal은 사진 인식을 제공하나 복합 요리 정확도가 낮아 수동 보정이 필요하다. 실제 20대 당뇨 환자 인터뷰 결과, 기존 앱 이탈의 핵심 원인은 "기록 노력 대비 실질적 피드백 부재"였다. On-Care는 YOLOv8(음식 여부 1차 필터) → Gemini Vision API(음식 종류·섭취량 분석) → 공공데이터 식품영양성분 DB 매핑의 3단계로 사진 한 장 완결 기록을 구현하고, 한국 음식 정확도 문제를 공공데이터로 해결한다.

#### ② RAG 기반 개인화 AI 코칭
경쟁 3사 모두 개인 건강 이력을 참조하는 AI 코칭이 없다. 인터뷰에서 확인된 바, 만성질환 환자는 ① 동일 식품 반복 섭취 시 대체 식품 정보, ② 혈당 급등·나트륨 과다 식품에 대한 맥락적 경고가 충족되지 않는다. On-Care는 LangChain + Pinecone + GPT-4o RAG 파이프라인으로 "내 이번 주 기록 기준" 맞춤 조언을 생성한다.

#### ③ 오프라인 운동 환경 연결
경쟁 3사 모두 헬스장·트레이너 연동 기능이 없어 사용자가 별도 채널로 이탈해야 한다. On-Care는 카카오맵 API 기반 헬스장 검색·예약·트레이너 인앱 채팅을 통합하고, 트레이너 연결 시 사용자의 누적 건강 데이터 요약본을 자동 전달해 초기 상담 효율을 높인다.

#### ④ 고혈압·당뇨 위험군 특화 설계
경쟁 앱들은 체중 감량 또는 일반 건강 관리 목적으로 설계되어 만성질환 관리 구조를 반영하지 못한다. On-Care는 서비스 기획 단계부터 나트륨 섭취 경고, GI(혈당지수) 기반 식품 분류, 야식·불규칙 식사 패턴 감지를 핵심 기능으로 설계하였다.

### 고객 및 Pain Point

**타겟**: 20~30대 만성질환(고혈압·당뇨·이상지질혈증) 위험군

**핵심 Pain Point (실제 20대 당뇨 환자 인터뷰 기반)**
- 기록 자체가 행동 변화로 이어지지 않아 앱 사용을 중단 — 기존 앱은 "기록 도구"에 그침
- 허용 식품군은 이미 알고 있으나, 반복 식단의 대체 식품 정보와 혈당 급등·나트륨 과다 식품에 대한 맥락적 경고가 없음
- 관리의 실제 핵심은 정밀한 영양 수치 추적이 아닌 누적 패턴 기반의 맥락적 조언

**Vitamin (있으면 좋은 것)**
- 게이미피케이션으로 건강 관리가 습관화되는 경험
- 트레이너에게 내 데이터가 자동 전달되어 반복 설명 불필요

### PMF (Product-Market Fit) 근거

국민건강보험공단 데이터 기준 20~30대 만성질환 유병률이 5년간 당뇨 38%, 고혈압 28%, 이상지질혈증 53% 증가하는 구조적 수요가 존재한다. 경쟁 제품 3사(Noom·Samsung Health·MyFitnessPal)는 모두 체중 감량 또는 일반 건강 관리 목적으로 설계되어 만성질환 위험군에 특화된 기능이 부재하다. 실제 환자 인터뷰를 통해 기존 앱이 실질적 행동 변화를 유도하지 못한다는 점이 실증되었으며, 이는 "기록 자동화 + 누적 이력 기반 맞춤 코칭 + 만성질환 특화 설계"라는 On-Care의 설계 방향이 충족되지 않은 시장 공백임을 뒷받침한다.
